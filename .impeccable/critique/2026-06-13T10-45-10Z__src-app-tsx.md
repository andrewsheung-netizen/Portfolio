## Anti-Patterns Verdict

Not AI slop. **CLI scan: 0 findings.** **In-page overlay: 1** — `flat-type-hierarchy` on the Activity/Summary views (computed sizes 12/14/16/18/22px, 1.8:1). The CLI (static markup) and in-page (live computed styles) disagree here because the flatness is a runtime property of those two views, which have no big mono anchor like the dashboard's net-worth display. Real but minor; arguably acceptable for a log/table register. No other tells: committed dark-teal identity, mono numerals, shadow-free, no eyebrows/gradient-text/side-stripes.

## What's Working

1. Every destructive or money-moving action is now both **logged and reversible** from one place — the Activity tab. Buys, sells, option closes, mortgage payments, and cash injections/withdrawals all carry a two-step Undo that atomically reverses the position/cash effect. This is a genuinely complete control story.
2. The **buy-funding model is honest**: "already owned" (no cash change), "buy from cash" (debits the balance), or "fresh injection" (net worth rises) — each produces the correct net-worth movement, verified live.
3. The **Summary** answers per-category questions over any range with correct HKD aggregation — Realized P&L, Net invested (bought vs sold), Mortgage interest/principal, and Capital flows — verified against seeded data to the dollar.

## Bugs found and fixed during this review

- **[was P1] "Net invested" sign was inverted.** The card rendered `-netInvested`, so net *buying* would display as a negative under a "Net invested" label — actively misleading. Fixed to show positive when capital is deployed, negative when net divested. Verified: net-divested test data now reads −HK$89,296 correctly.
- **[was P2] Active filter/range chip label was invisible.** A CSS cascade bug — the later `.toggle` rule overrode `.toggle-on`'s text colour, painting cyan text on a cyan fill. Fixed with `.toggle.toggle-on` specificity; active chip now reads dark-on-cyan at 11.2:1.

## Priority Issues (all P3; none blocking)

- [P3] Activity/Summary lack a typographic anchor (flat-type-hierarchy). Consider a summary headline figure or a larger section head so these views have the dashboard's instrument-panel presence.
- [P3] The transaction log has no search or pagination — fine now, but a multi-year history will get long. A date-range or text filter would scale it.
- [P3] An injection-funded buy shows "HK$0" cash impact in the log (correct — no cash moved — but could read "—" or "capital in").
- [P3] Carried over: no column sorting / ticker autofill / bulk; no retry button on failed refresh; no first-class shortcuts/help.

## Persona Red Flags

- Alex (power user): the log + filters + range presets are a big efficiency win; still wants sorting, ticker autofill, and log search as the history grows.
- Sam (screen reader/keyboard): tabs use aria-current, filters/ranges use aria-pressed, every undo is aria-live, dot colour is redundant with text. Solid. Tab switching is mouse/tap only (no arrow-key roving) — minor.
- Riley (stress tester): every event reverses cleanly (verified buy-undo reduces the holding, cash-flow-undo restores the balance); double-submit guarded; future dates blocked. Hard to break.
